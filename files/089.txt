Меня зовут Виктор Храпко. Это моя первая статья на Хабре. Прошу очень уж строго не судить.</p><p>Работаю на коммунальном предприятии и абоненты предприятия постоянно жалуются, что не могут получить информацию о задолженности и передать показания приборов учета.</p><p>Для решения проблемы уже был установлен личный кабинет на сайте, была установлена куча многоканальных телефонов, функционирует Viber бот.</p><p>Но абоненты продолжают жаловаться, что телефоны на отвечают или все время заняты, а в личном кабинете и Viber боте необходимо регистрироваться. Эта процедура почему то очень настораживает абонента.</p><p>Решил для эксперимента добавить еще одного чат-бота, где не требовалась бы регистрация абонента. На вопрос где его разместить тут же прозвучало Viber. Ну видимо он у всех на слуху. Но есть же еще и Telegram и FaceBook Мессенджер и Скайп и т.д.</p><p>Тут я вспомнил рассказ авиаконструктора Антонова. Они проектировали новый самолет на замену всем известному АН-2. В АН-2 всего девять мест. АН-14 был сделан на 10 мест. Комиссии очень понравился самолет, но экономисты что-то там посчитали и сказали, что нужно 11 мест для рентабельности. И тогда Антонов дал приказ конструировать самолет на 15 мест. Мол комиссия в следующий раз скажет — хорошо бы 13 мест. А мы скажем — а у нас 15 </p><p>Справедливости ради скажу, что в оригинале я такого самолета так и не увидел, но по моему даже летал на его чешской модификации Л-410. Вспомнить точно не могу. Было это более 30 лет тому назад.</p><p>Так вернемся к ботам. Я решил, что там где Viber там должны быть и другие мессенжеры.</p><p>И вопрос руководства о других мессенджерах будет закрыт так же как у Антонова.</p><p>Целью было составить все так, чтобы добавление нового чат бота в новом мессенджере занимало минимум программирования.</p><p>Встал выбор языка. Так как толком из языков программирования для Веб я знаю только PHP, то и думать долго не пришлось.</p><p>Как обычно начал работу с поиска в Гугл со словами чат бот PHP.</p><p>Благо Гугл не подвел и я собрал небольшой цикл статеек, которые мне здорово помогли</p><p><a href="https://timeweb.com/ru/community/articles/kak-sozdat-chat-bota-dlya-viber-1" rel="noopener noreferrer nofollow">По Viber боту</a></p><p><a href="https://imakebots.ru/article/registriruem-bota-u-botfather-ustanavlivaem-webhook" rel="noopener noreferrer nofollow">По Telegram</a></p><p><a href="https://habr.com/ru/post/281559/" rel="noopener noreferrer nofollow">Для FaceBook Мессенджера</a></p><p>К сожалению по Скайпу внятной информации не нашел. Только в одном месте нашел, что с 2019 года чат боты не регистрируются.</p><p>Ну да и бог с ним, решил я. Как-нибудь попробуем первые три.</p><p>Я не буду здесь описывать процесс регистрации самих ботов на сайтах этих мессенжеров.</p><p>Небольшая тонкость была у Telegram. Когда я пытался там подписаться на @BotFather, то я нашел два или три бота с одинаковым именем и иконкой. Но ответами, описанными в статье порадовал только один.</p><p>У FaceBook Мессенджера по-моему интерфейс разработчика меняется через день. Я нашел по моему четыре статьи и в каждой из них был показан другой интерфейс. Ну при известном опыте можно освоится.</p><p>Анализ показал, что разработка бота состоит из трех частей.  </p><p>Первая — Получить так называемый токен от мессенджера</p><p>Вторая — Указать сайту мессенджера где же в интернете располагается твой обработчик событий (webhook)  </p><p>Третья — собственно реализация самого чат бота.</p><p>Наиболее легко это выглядело у Телеграм бота</p><p>Токен получаем при создании бота</p><p>Выполняем из броузера команду  </p><p>https://api.telegram.org/botТОКЕН/setwebhook?url=Путь к боту</p><p>Если не ошибся с путем, то выдаст сообщение, где есть заветное слово OK</p><p>У Viberа уже нужно было разместить файл бота на сервере и выполнить специальный файлик. Он есть в первоисточнике, о котором я говорил выше. Результатом тоже должна быть строка содержащая OK.</p><p>Для FaceBook нужно было также разместить файл бота на сайте и из интерфейса FaceBook установить путь к WebHook.</p><p>В теле ботов для Viber и ФМ есть специальная ветка, отвечающая за регистрацию вебхука.</p><p>Итак приступаем к программированию.</p><p>Наша задача состоит в том, чтобы</p><ol><li><p>Прочитать данные, которые пришлет нам мессенжер.</p></li><li><p>Выделить из этих данных сам текст сообщения.</p></li><li><p>Создать ответ на полученное сообщение.</p></li><li><p>Отправить ответ.</p></li></ol><p>Для начала создадим несколько сервисных функций на PHP</p><p>Функция, которая загружает данные, полученные от мессенджера. Мы знаем что все мессенджеры используют для обмена JSON формат. Поэтому загружаем данные и для порядка и уверенности пишем их в лог файл. Имя файла является параметром данной функции. Также лог файл дает возможность полностью оценить структуру, которую присылает мессенджер.</p><pre><code class="php">function loadData($filename)
{
  $input = json_decode(file_get_contents('php://input'), true);
//Пишем в файл лог сообщений
  file_put_contents($filename, '$input: '.json_encode($input, JSON_UNESCAPED_UNICODE | JSON_PRETTY_PRINT)."\n", FILE_APPEND);
  return $input;
}</code></pre><p>Функция формирования ответа от чат-бота</p><pre><code class="php">function getAnswer($t)
{
// Ф-ция реализующая сам ответ.
// Я намеренно не привожу ее текст.
// Он зависит от поставленной перед вами задачи
// В простейшем случае можно ответить тем, что получили
  return $t;
}</code></pre><p>Функция отсылки данных на сервер мессенджера</p><p>Опять же пересылаем JSON на указанный адрес с помощью CURL</p><pre><code class="php">function sendCurl($url, $data)
{
  $request_data = json_encode($data);
//here goes the curl to send data to user
  $ch = curl_init($url);
  curl_setopt($ch, CURLOPT_POST, 1);
  curl_setopt($ch, CURLOPT_POSTFIELDS, $request_data);
  curl_setopt($ch, CURLOPT_HTTPHEADER, array('Content-Type: application/json'));
  $response = curl_exec($ch);
  $err = curl_error($ch);
  curl_close($ch);
  if($err) {return $err;}
  else {return $response;}
}</code></pre><p>Ну и функция, которая как бы собирает все вместе.</p><p>Что здесь происходит.</p><p>Мы передаем сюда имя файла лога $filename</p><p>Адрес по которому мы будем пересылать ответ $url</p><p>Входные данные $input</p><p>Три анонимные функции</p><p><strong>$getMessage</strong> — функция для получения самого текста из JSON структуры</p><p><strong>$changeFormat</strong> — функция форматирования текста ответа связанная с различным форматом сиволов для форматирования строки. Например Viber хочет три обратных кавычки для моноширинного шрифта, а остальные мессенджеры одну. Эта функция убирает лишние кавычки так как ответы у меня сделаны для Viber по умолчанию.</p><p><strong>$createSendData</strong> — функция формирования JSON структуры для ответа</p><p><br/>Так как ответ бота может быть довольно длинным или иногда хочется выделить часть ответа как отдельное сообщение, то в строку ответа вставляются разделительные знаки в виде ||.</p><p>Это разбивает строку ответа на несколько строк. И все они в цикле передаются на сайт мессенджера.</p><pre><code class="php">function sendAnswer($filename, $url, $input, $getMessage, $changeFormat, $createSendData)
{
  $message = $getMessage($input);
  if ($message &lt;&gt; '') {
    $text = getAnswer($message);
    $text = $changeFormat($text);
    do {
      $i = strpos($text, "||");
      if ($i == false) { $i = strlen($text); }
      $data = $createSendData($input, substr($text, 0, $i));
      file_put_contents($filename, '$answer: '.json_encode($data, JSON_UNESCAPED_UNICODE | JSON_PRETTY_PRINT)."\n", FILE_APPEND);
      $res = sendCurl($url, $data);
      file_put_contents($filename, '$res: '.json_encode($res, JSON_UNESCAPED_UNICODE | JSON_PRETTY_PRINT)."\n", FILE_APPEND);
      $text = substr($text, $i + 2);
    } while ($text &lt;&gt; '');
  }
}</code></pre><p>Я разместил все вышеприведенные функции в файл data.php. Каждый разработчик может называть файлы по своему. Никаких принципиальных требований нету. Просто в теле самого бота всегда присутствует строка include("data.php");</p><p>Теперь осталось реализовать сам бот</p><p>Пример для Telegram в файле telegram_bot.php</p><pre><code class="php">&lt;?php

include("data.php");

$token   = 'ВАШ ТОКЕН';
$url     = 'https://api.telegram.org/bot' . $token . '/sendMessage';
$logfile = 'telegram_in.txt';

$input = loadData($logfile);

$getMessage = function ($input) {
  return isset($input['message']['text']) ? $input['message']['text'] : '';
};

$changeFormat = function($text) {
  return str_replace('```', '`', $text);
};

$createSendData = function($input, $text) {
  $data['chat_id'] = $input['message']['chat']['id'];
  $data['parse_mode'] = 'markdown';
  $data['text'] = $text;
  return $data;
};

sendAnswer($logfile, $url, $input, $getMessage, $changeFormat, $createSendData);

?&gt;</code></pre><p>Что здесь есть. Несколько очевидных констант описывающих ваш токен, адрес куда слать ответ и имя лог файла.</p><p>Вызов функции loadData для загрузки JSON структуры от мессенджера.</p><p>И три анонимные функции о которых я говорил ранее.</p><p><strong>$getMessage</strong> — проверяет есть ли в полученной от мессенджера JSON структуре значение $input['message']['text']. Если есть, то оно и возвращается в качестве текста от мессенджера.
