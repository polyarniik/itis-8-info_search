        </a></div></div></div> <!-- --> <!-- --></div></div> <!-- --> <div class="tm-article-body" data-gallery-root="" lang="ru"><div></div> <div id="post-content-body"><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>В этой статье мы разберем один из способов оптимизации хранения данных и запросов, который поможет ускорить процесс выполнения задачи с помощью использования кодеков сжатия в колонках. И протестируем какие результаты можно получить при использовании кодеков.</p><p>В clickhouse есть несколько алгоритмов сжатия: LZ4 (по умолчанию), ZSTD, LZ4HC и экспериментальный DEFLATE_QPL. Подробнее про них можно прочитать в <a href="https://clickhouse.com/docs/en/sql-reference/statements/create/table/#specialized-codecs" rel="noopener noreferrer nofollow">документации</a>.</p><p>В clickhouse есть несколько кодеков для обработки данных:</p><ul><li><p>Delta(delta_bytes) — Метод сжатия, при котором необработанные значения заменяются разностью двух соседних значений, за исключением первого значения, которое остается неизменным.</p></li><li><p>DoubleDelta - Вычисляет дельту дельт и записывает ее в компактной двоичной форме.</p></li><li><p>Gorilla - Вычисляет XOR между текущим и предыдущим значением с плавающей запятой и записывает его в компактной двоичной форме.</p></li><li><p>FPC — Новый кодек сжатия для чисел с плавающей точкой, появился в версии 22.6.</p></li><li><p>T64 — Обрезает лишние биты для целых значений, включая типы даты/времени.</p></li></ul><details class="spoiler"><summary>А как оптимизировать хранение для строк?</summary><div class="spoiler__content"><p>Также есть способы оптимизации для строк, но с ними меньше вариантов, если в колонке со строками вариативность значений меньше 10 тысяч, то отличный эффект по скорости и сжатию даст тип колонки LowCardinality(String)</p><p><a href="https://clickhouse.com/docs/ru/sql-reference/data-types/lowcardinality/" rel="noopener noreferrer nofollow">https://clickhouse.com/docs/ru/sql-reference/data-types/lowcardinality/</a></p></div></details><h2>Какой кодек когда использовать?</h2><p>И второй вопрос, кодеки сжимают данные, но как влияют на скорость запросов?</p><p>С одной стороны применение кодека требует выполнять операцию по декодированию данных, с другой, сжатые данные могут занимать меньше места на диске, а следовательно операция чтения при запросе будет происходить быстрее.</p><p>Кодеки также дают разные результат на 32-битных и 64-битных числах, поэтому будем сравнить и разряд чисел.</p><p>Рассмотрим работу кодеков для нескольких типов распределения данных в таблице:</p><ul><li><p>rand_seq - cлучайная последовательность чисел;</p></li><li><p>const_seq - монотонно-возрастающая последовательность; </p></li><li><p>gauss_seq - гауссово распределение; </p></li></ul><p>Для следующих типов float32, float64, int32,  int64 и DateTime. Алгоритм сжатия  LZ4.</p><p>Заполним таблицу 100 млн. строк и сравним результаты с кодеками и без. Скорость выполнения запросов будем проверять таким запросом</p><pre><code class="sql">SELECT max(test_column) from test_table</code></pre><details class="spoiler"><summary>Код создания таблицы и кодеков для колонок</summary><div class="spoiler__content"><pre><code class="sql">create table test_table

В большинстве случаев int64 используется для хранения куда меньших чисел.</p><p>Если вам повезло и данные монотонно возрастающие то вариантов оптимизации много, в идеальном случае для int лучшее сжатие DoubleDelta, Delta оптимальнее по сжатию и скорости чтения. Для float лучший результат сжатия дает новый кодек FPC, но чтение будет медленнее.</p><p>Для распределения гаусса (например данные метрик) - для int64 и int32 лучшее сжатие и время выполнение обеспечивает кодек T64. Для float сжатие дает только новый кодек FPC</p><details class="spoiler"><summary>Как добавить кодек к существующей таблице?</summary><div class="spoiler__content"><pre><code class="sql">ALTER TABLE test_table MODIFY COLUMN column_a CODEC(ZSTD(2)); </code></pre><p>но работает только с новыми данными в таблицу, чтобы применить кодек к старым данным: </p><pre><code class="sql">ALTER TABLE test_table UPDATE column_a = column_a WHERE 1</code></pre><p></p></div></details><p></p></div></div></div> <!-- --> <!-- --></div> <!-- --> <!-- --></div> <!-- --> <div
