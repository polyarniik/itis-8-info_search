Статья будет полезна junior‑ и middle‑ разработчикам. Поехали!</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/2ef/18a/991/2ef18a99120354466523aa8eed8a152c.png" height="720" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/2ef/18a/991/2ef18a99120354466523aa8eed8a152c.png" width="1280"><figcaption></figcaption></img></figure><h3>Применение Next.js и почему возникает необходимость в увеличении скорости</h3><p>Большие изменения произошли в разработке веб‑приложений из‑за роста Next.js — фреймворка, помогающего создавать веб‑приложения используя JavaScript, не затрачивая свои силы на создание серверной инфраструктуры. Это значительно облегчает процесс создания гибридного приложения с отображением страниц как на стороне клиента, так и на стороне сервера. Несмотря на то что фреймворк достаточно простой, вопрос увеличения скорости приложений является актуальным.</p><p>Скорость приложения влияет на время загрузки страницы (предоставление клиенту кода приложения, данных и стилей) и как следствие на UX. В случае необходимости отправки дополнительных ресурсов сервером, таких как изображения, производительность снижается. Для избежания данных проблем рекомендуем несколько способов увеличения скорости приложений Next.js. </p><h3>Применение рендеринга на стороне сервера</h3><p>Популярная методика, используемая для отображения исходного HTML‑кода веб‑страницы на сервере перед ее отправкой на сторону клиента. Это поможет приложению сократить время, которое необходимо для рендеринга первой страницы на стороне клиента, что позволяет пользователю видеть содержимое страницы намного быстрее. Рендеринг на стороне сервера повышает производительность приложений, в особенности на мобильных устройствах.</p><p>При экспорте функции getServerSideProps (рендеринг на стороне сервера) со страницы, Next.js предварительно отображает эту страницу при каждом запросе, с использованием данных, возвращаемых getServerSideProps.</p><p><strong>Пример:</strong></p><pre><code class="javascript">export async function getServerSideProps(context) {
  return {
    props: {}, // will be passed to the page component as props
  };
}</code></pre><p>Рендеринг выполняется только на стороне сервера и никогда не запускается в браузере. Если страница использует getServerSideProps, то при запросе этой страницы на стороне клиента Next.js отправляет запрос API на сервер, который запускает его. Происходит извлечение данных из API и возврат полученных данных компоненту страницы в качестве атрибута.</p><p><strong>Пример:</strong></p><pre><code class="javascript">// This function will be called by the server

export async function getServerSideProps({ context }) {
  // Fetch data from external API

  const data = await fetch(YOUR_API);

  // Returning the fetched data

  return { props: { data } };
}



function SSRPage({ data }) {
  // Displaying the data to the client

  return &lt;div&gt;{data}&lt;/div&gt;;
}

export default SSRPage;</code></pre><p>В данном примере каждый раз при посещении пользователем страницы рендеринга на стороне сервера функция getServerSideProps будет вызываться сервером и возвращать полностью отображенную статическую страницу.</p><h3>Динамический импорт</h3><p>При начальной загрузке приложением загружаются все необходимые компоненты и CSS. Использование динамического импорта позволяет разбить код на фрагменты и загружать по необходимости. По мере использования клиентом приложения, компоненты, с которыми не взаимодействует клиент, не будут загружены. Значительно повышается производительность, в особенности на мобильных устройствах. Также это сокращает время начальной загрузки и объем подключаемого пакета данных. В случае, когда пользователь не вошел в систему, доступно отложить загрузку компонента входа.</p><p>Для использования динамического импорта необходимо импортировать код с помощью динамического импорта ES2020. Динамический импорт позволяет условно импортировать файлы когда они нам нужны, для этого необходимо вызвать ключевое слово import и передать ему путь к нужному файлу. Также данная функция возвращает обещание, и с ней возможно использовать .then и .catch.</p><p><strong>Пример:</strong></p><pre><code class="javascript">import dynamic from 'next/dynamic';
import SimpleButton from '../components/Buttons';

const DynamicComponent = dynamic(() =&gt; import('../components/LoginButton'));

function Program() {
  return (
    &lt;div&gt;
      &lt;SimpleButton /&gt;

      &lt;DynamicComponent /&gt;
    &lt;/div&gt;
  );
}

export default Program;</code></pre><p>В данном примере используется динамический компонент, предоставляемый фреймворком для динамической загрузки кнопки входа в систему. Существует возможность передачи имени компонента, массив имен модулей и функции внутри компонента, которые будут вызываться при загрузке модуля.</p><h3>Кэширование часто используемого содержимого</h3><p>Next.js имеет встроенное кэширование, по этой причине страницы загружаются гораздо быстрее. Кэширование снижает использование полосы пропускания за счет использования содержимого кэша вместо исходного источника, тем самым улучшая время отклика. Для реализации кэширования в приложении нужно вручную задать http‑заголовки для постраничных запросов. </p><p><strong>Пример:</strong></p><pre><code class="javascript">export default function handler(req, res) {
  res.setHeader('Cache-Control', 's-maxage=10');
}</code></pre><p><strong>Для рендеринга на стороне сервера:</strong></p><pre><code class="javascript">export async function getServerSideProps({ req, res }) {
  res.setHeader(
    'Cache-Control',
    'public, s-maxage=10, stale-while-revalidate=59'
  );
  return {
    props: {},
  };
}</code></pre><p>Next.js  автоматически добавляет статические файлы и ресурсы, нет необходимости добавлять кэширование вручную.</p><h3>Удаление неиспользуемых зависимостей</h3><p>Многие приложения зависят от сторонних пакетов. Все зависимости положительно влияют на приложение, но в то же время они увеличивают его размер и время загрузки. Если в Next.js используются пакеты npm, необходимо отслеживать неиспользуемые зависимости. Они занимают место в конечном пакете и могут вызвать неожиданную работу приложения.</p><p>Если проект небольшой, можно легко найти неиспользуемые зависимости и удалить их из package.json файла. В случае большого проекта с большим количеством различных зависимостей, возникают трудности в определении неиспользуемых. В данном случае используется depcheck‑npm.</p><p>Рекомендуется удалять зависимости поочередно — одну за другой, и перезапускать приложение после каждого удаления, чтобы убедиться в том что не нарушена работа приложения.</p><h3>Оптимизация изображений</h3><p>Оптимизация изображений заключается в уменьшении размера файла. Уменьшение размеров файлов изображений, являющихся одним из самых больших ресурсов, влияет на повышение производительности приложения.</p><p>Процесс проходит в 2 этапа: </p><ol><li><p>изменение изображения до меньшего размера; </p></li><li><p>сохранение в формате jpeg — для фотографий, png — для графики.</p></li></ol><p>Next.js имеет встроенный компонент next/image, который может использоваться вместо собственного &lt;img&gt; компонента.</p><p><strong>Пример: import Image from 'next/image'</strong></p><pre><code class="javascript">function OptimizedImage() {
  return (
    &lt;&gt;
      &lt;h1&gt;Next.js Image&lt;/h1&gt;

      &lt;Image
        src={image_url}
        alt="Any Text"
        width={500}
        height={500}
        blurDataURL="URL"
        placeholder="blur"
      /&gt;
    &lt;/&gt;
  );
}</code></pre><p>Преимущества next/image компонента:</p><ul><li><p>Отложенная загрузка - процесс загрузки изображения происходит только тогда, когда оно видно в окне клиента. По умолчанию компонент избирательно загружает изображения, что сокращает время загрузки. Если нет необходимости откладывать загрузку изображения, необходимо отключить priority={true}.</p></li><li><p>Изображения заполнители - используя next/image компонент, существует возможность добавить размытый заполнитель для любого изображения, используя placeholder атрибут.</p></li><li><p>Предварительная загрузка изображений - при наличии нескольких изображений на странице возможно установить приоритет загрузки с помощью next/image. </p></li></ul><h3>Оптимизация скриптов</h3><p>Дополнительно многие приложения используют сторонние скрипты, например Google Analytics, Google AdSense и др., они могут еще больше замедлять Next.js приложения. Вместо использования &lt;script&gt; тега по умолчанию, лучше использовать next/script компонент Next.js. Это позволит установить приоритет загрузки для сторонних скриптов.</p><p><strong>Пример:</strong></p><pre><code class="javascript">import Script from 'next/script';

export default function OptimizedScript() {
  return (
    &lt;&gt;
      &lt;Script
        id="YOUR_ID"
        src="URL"
        onError={(err) =&gt; {
          console.error('Error', err);
        }}
        onLoad={() =&gt; {
          // Function to perform after loading the script
        }}
      /&gt;
    &lt;/&gt;
  );
