        </a></div></div></div> <!-- --> <!-- --></div></div> <!-- --> <div class="tm-article-body" data-gallery-root="" lang="ru"><div></div> <div id="post-content-body"><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><figure class="full-width"><img data-blurred="true" data-src="https://habrastorage.org/getpro/habr/upload_files/f6e/681/fd2/f6e681fd23da9c4d75bf41a0361a2b8a.jpg" height="663" src="https://habrastorage.org/r/w780q1/getpro/habr/upload_files/f6e/681/fd2/f6e681fd23da9c4d75bf41a0361a2b8a.jpg" width="1158"><figcaption></figcaption></img></figure><p>В этой статье я покажу, как организовать простейший автодеплой на сервере. Для автодеплоя через Bitbucket Webhooks и PHP не нужно использовать какие-то сложные решения.</p><p>Для начала можно подумать, что <code>git pull</code>  выполняется через PHP с помощью этого куска кода:</p><pre><code class="bash">exec('cd /путь/к/корневой/папке &amp;&amp; git pull origin master');</code></pre><p>Но во время выполнения этого кода возникает ошибка доступа, даже, если SSH-ключ вашего текущего пользователя добавлен в Bitbucket.</p><p>Когда мы запускаем  <code>git pull</code> с помощью PHP, команда выполняется не от имени вашего  пользователя. Поэтому Bitbucket не распознает наш SSH-ключ. Из-за этого возникает ошибка доступа, если репозиторий закрытый. </p><p>Функция <code>exec</code> выполняет команды от имени стандартного пользователя вашего веб-сервера. В Nginx это обычно <code>www-data</code>. Самый простой способ узнать пользователя от имени которого выполняются PHP скрипты, выполнять этот кусок кода и смотреть в браузере:</p><pre><code class="php">$result = exec('whoami');
var_dump($result);</code></pre><p>После того, как вы узнали вашего пользователя, выполняем следующие шаги:</p><ol><li><p><strong>Генерируем SSH-ключ для этого пользователя</strong></p></li></ol><pre><code class="bash">sudo -u имя_этого_пользователя ssh-keygen -t rsa</code></pre><p>Путь, где лежит папка .ssh и сгенерированный ключ будет показываться во время генерации ключа. </p><ol start="2"><li><p><strong>Добавляем </strong><a href="http://bitbucket.org" rel="noopener noreferrer nofollow"><strong>bitbucket.org</strong></a><strong> в known_hosts</strong></p></li></ol><pre><code class="bash">ssh-keyscan -H bitbucket.org &gt;&gt; /путь/где/лежит/папка/.ssh/known_hosts</code></pre><ol start="3"><li><p><strong>Добавляем наш публичный ключ для этого пользователя в Bitbucket</strong></p></li></ol><p>Для начала выводим ключ терминале, копируем</p><pre><code class="bash">cat /путь/где/лежит/папка/.ssh/id_rsa.pub</code></pre><p>и добавляем в Bitbucket —  Personal settings-&gt;SSH keys-&gt;Add key.</p><ol start="4"><li><p><strong>Создаем Webhook</strong></p></li></ol><p>Заходим в репозиторий нашего проекта, потом в Repository settings-&gt;Webhooks-&gt;Add Webhook.</p><p>Прописываем название вебхука, ссылку на которую он должен триггериться и событие, при котором должен произойти автодеплой.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/bd0/d17/a85/bd0d17a85a417f29947bf37cc8aca2f6.png" height="669" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/bd0/d17/a85/bd0d17a85a417f29947bf37cc8aca2f6.png" width="703"/><figcaption></figcaption></figure><p>Нужно обратить внимание, что вебхук отправляет POST запрос. Если ваш URL настроен на GET запросы, вебхук не будет работать. Bitbucket не показывает ошибку, если ссылка настроена на GET запросы, но и деплой не происходит.</p><p>PHP скрипт можно дополнять по необходимости. Можно кроме выполнения <code>git pull</code> ещё мигрировать базу и выполнять другие команды. Вебхуки передают данные во время обращения к вашей ссылке. Эти данные вы можете обработать и сделать необходимые проверки.</p><h2>Ошибки, которые могу возникнуть</h2><p><strong>Ошибка #1</strong></p><p>Если владелец ваших файлов проекта пользователь <code>root</code>,  тогда Git может выдать такую ошибку "fatal: detected dubious ownership in repository".</p><p>Помогает рекурсивное изменение владельца папки на пользователя, от имени которого выполняются PHP скрипты.</p><pre><code class="bash">chown -R имя_вашего_пользователя:root /путь/к/корневому/директорию/проекта</code></pre><p>Еще можно в конфигах Git добавить эту настройку:</p><pre><code>[safe]
    directory = /путь/к/корневому/директорию/проекта</code></pre><p>Для этого нужно создать <code>.gitconfig</code> файл в домашнем директории вашего пользователя и в нем прописать эти настройки. Это тот директорий, где лежит ваша недавно сгенерированная папка <code>.ssh</code>. Эти настройки будут применяться, когда Git запустится от имени этого пользователя.</p><p><strong>Ошибка #2</strong></p><p>Ещё может возникнуть ошибка "fatal: empty ident name not allowed"</p><p>Для решения ошибки Git предлагает выполнить эти команды, чтобы добавить ваше имя и почту:</p><pre><code class="bash">git config --global user.email ""
git config --global user.name ""</code></pre><p>Но эти команды задают глобальные настройки, а нам нужно задавать настройки именно для нашего пользователя. Для этого мы заходим в уже созданный для нашего пользователя <code>.gitconfig</code> файл и добавляем следующие настройки:</p><pre><code>[user]
        name = ваше_имя
        email = ваша_почту</code></pre><p>После этого команда <code>git pull</code> со стороны PHP должна работать.</p><p></p></div></div></div> <!-- --> <!-- --></div> <!-- --> <!-- --></div> <!--
