успешное прохождение сценария, затем негативные тесты с невалидными и/или недопустимыми данными, null для обязательных ключей и т.д..</p><p>При выполнении каждого запроса API нам необходимо и важно проверить:</p><ul><li><p>корректность статуса кода;</p></li><li><p>полезную нагрузку ответа;</p></li><li><p>сам ответ от сервера: сообщение, ключ, значение, тип данных, структуру объектов;</p></li><li><p>заголовки ответа от сервера;</p></li><li><p>авторизацию с использованием методов аутентификации;</p></li><li><p>протокол HTTP / HTTPS согласно спецификации;</p></li><li><p>дополнительно доступы разных ролей, время ответа API, стиль формата обмена данными.</p></li></ul><hr/><p>Сейчас я хочу продемонстрировать  пример тест-кейса "Добавить нового питомца в магазин (Вызов метода POST)", используя сайт <a href="https://petstore.swagger.io/" rel="noopener noreferrer nofollow"><strong><u>https://petstore.swagger.io/</u></strong></a><strong> </strong></p><figure class="bordered full-width"><img alt='Тест-кейс API "Добавление нового питомца в магазин"' height="963" src="https://habrastorage.org/getpro/habr/upload_files/d52/bbd/ccc/d52bbdccc71c691c0a1a9e2851851639.JPG" title='Тест-кейс API "Добавление нового питомца в магазин"' width="1902"><figcaption>Тест-кейс API "Добавление нового питомца в магазин"</figcaption></img></figure><p><strong>Важно выполнить следующие шаги:</strong></p><div><div class="table"><table><tbody><tr><td><p align="left">Отправить POST запрос <a href="https://petstore.swagger.io/v2/pet" rel="noopener noreferrer nofollow"><u>https://petstore.swagger.io/v2/pet</u></a><u> </u>с телом:</p></td></tr></tbody></table></div></div><pre><code>{
    "id": 10,
    "category": {

    },

    "photoUrls": [
