VPS - что такое и где искать</h2><blockquote><p><strong>VPS</strong> (<em>virtual private server</em>) или <strong>VDS</strong> (<em>virtual dedicated server</em>), <strong>виртуальный выделенный сервер</strong> — услуга предоставления серверных мощностей провайдером.</p></blockquote><p>Существует множество сервисов по подбору VPS с конкретными характеристиками, вот топ-5 популярных провайдеров (взято из <a href="https://habr.com/ru/post/686238/" rel="noopener noreferrer nofollow">этой</a> статьи). Сам я использовал сервис timeweb.</p><ul><li><p><a href="https://timeweb.com/ru/" rel="noopener noreferrer nofollow">Timeweb</a></p></li><li><p><a href="https://beget.com/ru" rel="noopener noreferrer nofollow">Beget</a></p></li><li><p><a href="https://mchost.ru/" rel="noopener noreferrer nofollow">Mchost</a></p></li><li><p><a href="https://masterhost.ru/" rel="noopener noreferrer nofollow">Masterhost</a></p></li><li><p><a href="https://justhost.ru/" rel="noopener noreferrer nofollow">JustHost</a></p></li></ul><a class="anchor" id="part2" name="part2"></a><h2>2. Что понадобится</h2><ul><li><p>Для начала, сам <strong>выделенный сервер, а также домен</strong>. В данной статье я буду использовать VPS на Ubuntu 22.04. Важно отметить, что для ботов на webhooks telegram требует SSL-сертификат. Для собственных целей я приобрёл его, однако если вы не хотите тратиться, вы можете использовать <a href="https://core.telegram.org/bots/self-signed" rel="noopener noreferrer nofollow">self-signed certificate</a>.</p></li><li><p>Затем для общения с ним нам нужен <strong>клиент удалённого доступа</strong>. Я использовал <a href="https://www.putty.org/" rel="noopener noreferrer nofollow">PuTTY</a></p></li><li><p>Чтобы передать на сервер файлы нашего бота, потребуется FTP-клиент. Мой выбор пал на <a href="https://filezilla.ru/" rel="noopener noreferrer nofollow">FileZilla</a></p></li><li><p>Также для создания исполняемого <code>.jar</code> файла на своей машине мне понадобился Maven для терминала</p></li></ul><a class="anchor" id="maven_install" name="maven_install"></a><details class="spoiler"><summary>Установка Maven</summary><div class="spoiler__content"><p>Обновляем списки пакетов из репозиториев</p><pre><code>$ sudo apt update</code></pre><p>Устанавливаем maven</p><pre><code>$ sudo apt install maven</code></pre><p>Проверяем результат</p><pre><code>$ mvn -version</code></pre><p></p></div></details><h2>3. Конфигурация и application.properties</h2><a class="anchor" id="part3" name="part3"></a><p>Конфигурация моего Spring Boot приложения состоит из трёх файлов: конфигурация Spring, самого бота и файла <code>application.properties</code></p><details class="spoiler"><summary>application.properties</summary><div class="spoiler__content"><pre><code class="bash">telegram.bot-name=[имя вашего бота]
telegram.bot-token=[токен, который можно взять у BotFather]
telegram.webhook-path=[ваш домен в формате https://{вашДомен}:{порт}/bot{токенБота}]
server.port=[порт подключения]</code></pre><p></p></div></details><p>На данном этапе нужно отметить, что telegram поддерживает порты <strong>443, 80, 88 и 8443</strong>.</p><details class="spoiler"><summary>BotConfig.java</summary><div class="spoiler__content"><pre><code class="java">import lombok.Getter;

/* lombok - это плагин компилятора, который добавляет в Java 
новые «ключевые слова» и превращает аннотации в Java-код,
уменьшая усилия на разработку и обеспечивая некоторую 
дополнительную функциональность. */

import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Configuration;

@Configuration // помечаем класс, как конфигурацию
@Getter // устанавливаем геттеры каждому полю класса
public class BotConfig {

    /* с помощью @Value получаем значения из файла application.properties
  и устанавливаем их в соответствующие поля*/

    @Value("${telegram.bot-name}")
    String botName;

    @Value("${telegram.bot-token}")
    String token;

    @Value("${telegram.webhook-path}")
    String webhookPath;

}</code></pre><p></p></div></details><details class="spoiler"><summary>SpringConfig.java</summary><div class="spoiler__content"><pre><code class="java">import lombok.AllArgsConstructor;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.telegram.telegrambots.meta.api.methods.updates.SetWebhook;
import something.MessageHandler;
import something.MyBot;

@Configuration
@AllArgsConstructor // вносим конструктор для всех параметров
public class SpringConfig {
  
    private final BotConfig telegramConfig;

    @Bean
    public SetWebhook setWebhookInstance() {
        return SetWebhook.builder()
          .url(telegramConfig.getWebhookPath()).build();
    }

    @Bean
    public MyBot springWebhookBot(SetWebhook setWebhook,
                                  MessageHandler messageHandler) {
        MyBot bot = new MyBot(setWebhook, messageHandler);

        bot.setBotPath(telegramConfig.getWebhookPath());
        bot.setBotUsername(telegramConfig.getBotName());
        bot.setBotToken(telegramConfig.getToken());

        return bot;
    
    }
  
}</code></pre><p></p></div></details><h2>4. Процесс выгрузки</h2><a class="anchor" id="part4" name="part4"></a><h4>Настраиваем окружение</h4><p>Открываем PuTTY и подключаемся к нашему серверу. Все требуемые данные для входа должен предоставить провайдер после аренды VPS. </p><p>Вводим (1) IP в графу <em>Host Name</em> и жмём (2) <em>Open</em>.</p><figure class="full-width"><img alt="Вход в PuTTY" data-blurred="true" data-src="https://habrastorage.org/getpro/habr/upload_files/2ff/1d0/436/2ff1d0436f67a031898426423d2ff22f.jpg" height="535" src="https://habrastorage.org/r/w780q1/getpro/habr/upload_files/2ff/1d0/436/2ff1d0436f67a031898426423d2ff22f.jpg" title="Вход в PuTTY" width="592"/><figcaption>Вход в PuTTY</figcaption></figure><p>Далее входим под нашим юзером в систему и начинаем устанавливать нужные нам пакеты</p><p>Для начала обновляемся</p><pre><code>$ sudo apt update</code></pre><p>Устанавливаем нужный пакет <code>JRE (Java Runtime Environment)</code>. Я использую 17-ю версию Java, поэтому пишу следующую команду</p><pre><code>$ apt install openjdk-17-jre-headless</code></pre><p>Также нам потребуется утилита screen - консольная утилита позволяющая в действующей SSH сессии открывать неограниченное количество независимых виртуальных терминалов. Это нужно для того, чтобы после завершения удалённого подключения наш бот <s>не решил, что отношения на расстоянии это не для него</s> продолжал свою работу. Она может быть установлена по дефолту, однако если это не так:</p><pre><code>$ apt install screen</code></pre><p>Все нужные команды для работы с этой утилитой можно найти <a href="https://winitpro.ru/index.php/2021/02/05/ispolzovanie-utility-screen-v-linux/" rel="noopener noreferrer nofollow">здесь</a>.</p><h4>Последние штрихи и перенос на сервер</h4><p>На нашем устройстве идём в директорию проекта. Тут нам нужно создать исполняемый <code>.jar</code> файл (для этого нужно установить <a href="#maven_install" rel="noopener noreferrer nofollow">Maven</a>). Пишем следующее:</p><pre><code>$ maven clean install</code></pre><p>После этого в папке <em>botDirectory/target</em> должны появиться два файла: <code>BOTNAME-0.0.1-SNAPSHOT.jar</code> и <code>BOTNAME-0.0.1-SNAPSHOT.jar.original</code></p><p>Переходим в FileZilla. Вводим IP, имя пользователя, пароль, порт <strong>22</strong> и нажимаем <em>Quickconnect</em>. Теперь полностью переносим директорию с приложением в root, чтобы получилось вот так:</p><figure class=""><img alt="Вид корневой папки сервера" data-blurred="true" data-src="https://habrastorage.org/getpro/habr/upload_files/fe0/fe3/ff8/fe0fe3ff806df85de9916c2d1849aa3b.jpg" height="133" src="https://habrastorage.org/r/w780q1/getpro/habr/upload_files/fe0/fe3/ff8/fe0fe3ff806df85de9916c2d1849aa3b.jpg" title="Вид корневой папки сервера" width="203"/><figcaption>Вид корневой папки сервера</figcaption></figure><h4>Передаём адрес telegram'y</h4><p>Далее нужно сообщить телеграму, что наш бот будет работать через Webhook. Для этого в адресной строке вводим запрос:</p><pre><code>https://api.telegram.org/bot{ТокенБота}/setWebhook?url={АдресСвоегоWebhook}</code></pre><h4>Запускаем бота на сервере</h4><p>Заходим в PuTTY и выполняем такую команду.
